📢 Guía de Mensajes y Alertas del Sistema

Lefarma utiliza un sistema de notificaciones visuales para informarte sobre el éxito de tus operaciones o advertirte cuando algo no salió como se esperaba. Esta guía te ayudará a entender qué significan estos mensajes y cómo actuar ante ellos.

🎨 Tipos de Alertas

Las alertas aparecen en la esquina superior derecha de la pantalla con diferentes colores:

  • Verde (Éxito): Tu operación se guardó correctamente. No requiere acción adicional.
  • Naranja (Advertencia): Los datos se guardaron, pero hubo un detalle menor (ej: "No se pudo enviar el correo de notificación, pero la orden se creó").
  • Rojo (Error): La operación falló. El sistema te indicará el motivo específico.

🛑 Errores Comunes y Soluciones

1. Errores de Validación (Datos Incorrectos)

Son los más frecuentes. El sistema bloquea el guardado para proteger la integridad de la información.

Mensaje¿Qué significa?¿Qué hacer?
"El registro ya existe"Estás intentando crear algo (RFC, Nombre, Clave) que ya está en el sistema.Busca el registro existente para editarlo en lugar de crear uno nuevo.
"Campo obligatorio"Dejaste un dato vacío que el sistema necesita.Llena todos los campos marcados con un asterisco (*).
"Formato inválido"Escribiste algo mal (ej: un correo sin @ o un RFC mal estructurado).Revisa la ortografía y el formato del dato ingresado.

2. Errores de Permisos

Si ves un mensaje que dice "No tienes permisos suficientes" o "Acceso Denegado":

  • Tu usuario no tiene asignado el rol necesario para esa acción específica.
  • Solución: Contacta al administrador del sistema para revisar tu perfil de seguridad.

3. Errores de Conexión o del Servidor

Si aparece un mensaje genérico como "Error interno del servidor (500)" o "No se pudo conectar con la API":

  • Puede ser un fallo temporal en la red o un problema técnico en el servidor.
  • Solución: Espera un momento y vuelve a intentarlo. Si el problema persiste, reporta el error.

🛠️ Cómo Reportar un Problema

Si encuentras un error que no puedes resolver, por favor incluye la siguiente información al contactar a Soporte Técnico:

  1. Pantallazo: Una captura del error donde se vea el mensaje rojo.
  2. Acción: ¿Qué estabas intentando hacer? (ej: "Estaba intentando autorizar la orden #123").
  3. Hora: El momento exacto en que ocurrió.
Sugerencia: Si el sistema te muestra un código largo de números y letras (ID de Petición), cópialo y envíalo. Eso ayuda a los programadores a encontrar tu caso en milisegundos.

💡 Consejos de Oro

  • Lee antes de cerrar: Las alertas rojas suelen decir exactamente qué campo está mal. No las cierres sin leerlas primero.
  • Refrescar: A veces, simplemente refrescando la página (F5) se solucionan problemas de conexión momentáneos.
