⚙️ Motor de Workflows y Flujos de Autorización

El módulo de Workflow es el motor de estados de Lefarma. Permite definir flujos de trabajo dinámicos para procesos de negocio (como la aprobación de Órdenes de Compra), permitiendo configurar pasos, acciones, condiciones de ruteo dinámico y notificaciones automatizadas sin necesidad de cambios en el código.

🏗️ Arquitectura del Motor

El motor funciona como una máquina de estados finitos basada en los siguientes componentes:

  • Workflow: El contenedor principal ligado a un proceso (ej: ORDEN_COMPRA).
  • Pasos (Steps): Estados individuales del proceso (ej: PENDIENTE_GERENTE, AUTORIZADA).
  • Acciones (Transitions): Botones o eventos que mueven el proceso de un paso a otro (ej: Autorizar, Rechazar).
  • Condiciones (Routing): Lógica IF/THEN que determina a qué paso saltar según datos del negocio (ej: Si Total > 5000 saltar a Paso 5).
  • Participantes: Roles o Usuarios específicos que tienen permiso para ejecutar acciones en un paso determinado.

🌐 API Endpoints

Base URL: /api/config/Workflows

Gestión de Estructura

  • POST /: Crea un nuevo encabezado de workflow.
  • PUT /{idWorkflow}/pasos/{idPaso}: Actualiza la configuración de un paso.
  • POST /{idWorkflow}/pasos/{idPaso}/acciones: Añade una transición de salida.
  • POST /{idWorkflow}/pasos/{idPaso}/condiciones: Define lógica de ruteo dinámico.

Ejemplo Request (Crear Acción):

{
  "nombreAccion": "Aprobar Gerencia",
  "tipoAccion": "APROBACION",
  "claseEstetica": "success",
  "idPasoDestino": 5
}

private readonly IWorkflowEngine _engine;

public async Task ProcesarAccion(int ordenId, int accionId, int usuarioId) {
    var result = await _engine.ExecuteTransitionAsync(ordenId, accionId, usuarioId);
    if(result.Success) {
        // La orden cambió de estado correctamente
    }
}

🛑 Validaciones del Motor

  1. Paso Único de Inicio: Solo puede existir un paso marcado como EsInicio.
  2. Caminos Finales: El sistema valida que todos los caminos lleguen eventualmente a un paso EsFinal.
  3. Permisos: Antes de ejecutar una acción, el motor valida que el UsuarioId tenga el RolId configurado en la tabla de Participantes para ese paso.
