📂 Módulo de Catálogos Generales

El módulo de Catálogos es el núcleo de configuración de Lefarma. Proporciona la infraestructura de datos maestros necesaria para el funcionamiento de los procesos operativos, financieros y de recursos humanos. Está diseñado bajo una arquitectura modular que permite gestionar entidades clave de manera centralizada.

🎯 Propósito del Sistema

El sistema de catálogos permite:

  • 🏢 Estructura Organizacional: Definir la jerarquía de Empresas, Sucursales y Áreas.
  • 💰 Configuración Financiera: Gestionar Bancos, Formas de Pago, Medios de Pago y Cuentas Contables.
  • 📦 Logística y Operaciones: Administrar Unidades de Medida y Proveedores.
  • 📝 Reglas de Negocio: Configurar Tipos de Gasto y Estatus de Órdenes.

Características principales:

  • CRUD completo con validaciones del lado del servidor (FluentValidation) y cliente (Zod).
  • Normalización automática de nombres (Upper Case) para búsquedas consistentes.
  • Sistema de borrado lógico (campo Activo).
  • Auditoría básica (FechaCreacion, FechaModificacion).
  • Integración con componentes de tablas avanzadas con filtrado y paginación.

🌐 API Endpoints

Todos los catálogos siguen un estándar de rutas: /api/catalogos/[entidad].

Ejemplo: Empresas

  • GET /api/catalogos/empresas: Obtiene el listado completo.
  • POST /api/catalogos/empresas: Crea una nueva empresa.
  • PUT /api/catalogos/empresas/{id}: Actualiza una empresa existente.
  • DELETE /api/catalogos/empresas/{id}: Borrado lógico de la empresa.

🔧 Backend: Cómo Usar

1. Inyectar Servicio de Catálogo

public class FacturacionService {
    private readonly IEmpresaService _empresaService;
    
    public FacturacionService(IEmpresaService empresaService) {
        _empresaService = empresaService;
    }
    
    public async Task Procesar() {
        var empresas = await _empresaService.GetAllAsync();
    }
}
