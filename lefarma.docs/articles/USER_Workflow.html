⛓️ Configuración de Flujos de Trabajo (Workflows)

El módulo de Workflows es una herramienta avanzada para administradores que permite diseñar cómo fluye la información y quién debe autorizar cada etapa de un proceso (como una compra o un pago) sin programar código nuevo.

📍 ¿Cómo Acceder?

Menú Principal > Configuración > Workflows

> 📸 [CAPTURAR PANTALLA: Listado de Workflows resaltando el botón "+ Nuevo Workflow"]

🚀 Cómo Crear un Flujo desde Cero

1. Definir el Encabezado

Haz clic en + Nuevo Workflow. Dale un nombre (ej: "Aprobación de Órdenes") y un Código de Proceso único (ej: ORDEN_COMPRA). Este código es el que vincula el flujo con la pantalla de operación real.

2. Diseñar los Pasos

Un flujo se compone de "Pasos". Cada paso representa una etapa del proceso:

  • Paso Inicial: Donde comienza el trámite (ej: "Captura de Orden").
  • Pasos Intermedios: Etapas de revisión (ej: "Validación CxP", "Autorización Gerente").
  • Pasos Finales: Donde termina el proceso (ej: "Autorizada", "Cancelada").

3. Configurar Acciones (Botones)

En cada paso, debes definir qué botones verá el usuario. Por ejemplo, en el paso de "Revisión", puedes agregar dos acciones:

  1. Autorizar: Mueve el flujo al siguiente paso.
  2. Rechazar: Regresa el flujo al paso anterior o lo cancela.

> 📸 [CAPTURAR PANTALLA: Ventana de edición de pasos y sus acciones]

4. Asignar Responsables (Participantes)

Para cada paso, debes indicar quién tiene permiso de presionar los botones. Puedes asignar roles completos (ej: "Gerentes de Área") o usuarios específicos.

🧠 Lógica Inteligente (Condiciones)

Puedes hacer que el flujo tome decisiones automáticas. Por ejemplo:

Ejemplo: Si el monto de la compra es mayor a $10,000, el flujo debe saltar automáticamente al paso de "Autorización Dirección", saltándose la validación básica.

📝 Guía de Estados Visuales

Clase EstéticaUso Sugerido
successBotones de Aprobación o Finalización.
dangerBotones de Rechazo o Cancelación.
warningBotones para solicitar cambios o devoluciones.
primaryAcciones de paso estándar o movimiento neutral.

💡 Consejos para Administradores

  • Diagrama Visual: Usa la opción "Ver Diagrama" para visualizar el mapa del flujo y detectar si algún paso quedó "huérfano" (sin salida).
  • Versiones: Si haces cambios grandes en un flujo activo, incrementa el número de versión para mantener el orden.
  • Notificaciones: Recuerda configurar los correos automáticos en la pestaña de notificaciones de cada acción para que los usuarios sepan cuando tienen algo pendiente por firmar.
