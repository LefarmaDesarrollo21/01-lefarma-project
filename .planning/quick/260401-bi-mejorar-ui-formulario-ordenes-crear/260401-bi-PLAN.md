# Quick Task 260401-bi: Mejorar UI formulario órdenes de compra

**Task:** Reorganizar y mejorar UI del card 'Datos Generales' y formularios en página de creación de órdenes
**Date:** 2026-04-01
**Status:** Ready for execution

## Problem Statement

El formulario actual de creación de órdenes tiene problemas de UX:
1. **Grid de 3 columnas muy denso**: 6 campos en filas de 3 se ven comprimidos y dificultan la lectura
2. **Jerarquía visual pobre**: No hay separación clara entre grupos de campos relacionados
3. **Placeholders genéricos**: Todos los selects dicen "Selecciona..." sin contexto
4. **Labels sin claridad**: Falta indicar qué campos son dependientes (Sucursal/Área filtran por Empresa)

## Proposed Improvements

### 1. Datos Generales - Layout Redesign
- **Cambiar a 2 columnas** en lugar de 3 para mejor legibilidad
- **Agrupar campos por lógica de negocio**:
  - Grupo 1: Ubicación (Empresa, Sucursal, Área) - 3 columnas en desktop
  - Grupo 2: Transacción (Tipo de Gasto, Forma de Pago) - 2 columnas
  - Grupo 3: Temporal (Fecha Límite) - 1 columna ancha
- **Mejorar placeholders** con contexto específico
- **Agregar descripciones** cuando sea necesario

### 2. Datos del Proveedor - Layout Redesign
- **Reorganizar campos fiscales**: Razón Social full-width primero
- **Grid de 3 para RFC, CP, Régimen** cuando aplique
- **Checkbox "Sin Datos Fiscales"** con mejor styling

### 3. Partidas - Card Styling
- **Mejorar visual del header** con más espaciado
- **Destacar el total de la partida** con mejor styling
- **Mejorar grid de inputs** de impuestos

### 4. Resumen - Visual Hierarchy
- **Alinear valores a la derecha** para mejor lectura
- **Mejorar visual del total** con más prominencia
